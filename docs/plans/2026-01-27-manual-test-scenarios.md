# 수동 테스트 시나리오 (MVP)

> 작성일: 2026-01-27
> 목적: 저장 → 목록 → 복사 흐름과 우클릭 저장 흐름을 빠르게 검증

## 0) 사전 준비
- `chrome://extensions`에서 확장을 "새로고침"
- 지원 사이트 중 하나를 연다
  - `https://chat.openai.com/`
  - `https://claude.ai/`
  - `https://gemini.google.com/`
  - `https://notebooklm.google.com/`

## 1) 팝업 저장/목록/삭제
1. 확장 팝업을 연다
2. "프롬프트 추가"에 아래를 입력하고 저장
   - 제목: `테스트 번역`
   - 내용: `영어로 번역: Hello`
3. 기대 결과
   - 목록 최상단에 `테스트 번역`이 보인다
4. 방금 저장한 항목에서 "삭제" 클릭
5. 기대 결과
   - 목록에서 해당 항목이 사라진다

## 2) 복사 성공 케이스
1. 팝업에서 방금 프롬프트의 "복사" 클릭
2. 기대 결과
   - 토스트에 "클립보드에 복사했습니다."가 표시된다
   - 입력창에 붙여넣기 시 텍스트가 들어간다

## 3) 복사 실패 케이스 (클립보드 접근 실패)
1. 팝업에서 "복사" 클릭
2. 기대 결과
   - 토스트에 복사 실패 안내가 나온다

## 4) 우클릭 선택 저장 + 최근 저장 강조
1. 사이트에서 문장 몇 줄을 드래그로 선택한다
2. 우클릭 → `Prompt Pocket에 저장` 클릭
3. 팝업을 연다 (열려 있다면 그대로 관찰)
4. 기대 결과
   - 상단에 저장 완료 토스트가 잠깐 표시된다
   - 방금 저장된 항목이 연두색으로 강조된다

## 관찰 포인트 / 알려진 한계
- 클립보드 접근은 브라우저 정책/환경에 따라 실패할 수 있다
- 우클릭 저장은 브라우저 컨텍스트 메뉴 정책에 영향을 받을 수 있다
